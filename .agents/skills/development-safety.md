# Development Safety Rules

## 【絶対厳守ルール】

1. **git checkout、git reset、git clean は絶対に使用禁止**
   - これらのコマンドは未コミットの変更を破壊するため、絶対に使用しない

2. **ファイルを変更する前に、必ず変更内容をユーザーに説明して承認を得る**
   - 変更前後に「これから○○のファイルを△△のように変更します。よろしいですか？」と必ず確認する

3. **バグ修正の際も、既存ファイルを上書きせず、修正案を提示するだけにする**
   - 修正案のコードを提示し、ユーザーが承認してから初めて変更する

4. **Gitの古いバージョンに戻す操作は一切しない**
   - `git checkout HEAD -- file`、`git revert` などの操作は禁止

5. **不明な点があれば必ず質問する**
   - 勝手な判断で操作しない

## 【推奨行動】

### 変更前の宣言
```
これから scripts/example.js の line 123-130 を以下のように変更します：
[変更内容の説明]

よろしいですか？
```

### 複数ファイルの変更時
- 1ファイルずつ承認を取る
- 前のファイルの変更が完了してから次のファイルについて尋ねる

### 破壊的な操作が必要な場合
代替案を3つ提示する：
1. 案A：[説明]
2. 案B：[説明]
3. 案C：[説明]

どの案で進めますか？

## 【禁止コマンドリスト】

- `git checkout`（ファイル名指定）
- `git reset --hard`
- `git clean`
- `git revert`
- `cp` による上書き（ユーザー明示的な承認なし）

## 【許可される操作】

- `git diff`（確認のみ）
- `git status`（確認のみ）
- `git log`（確認のみ）
- テキストエディタでの新しいファイル作成
- ユーザーの明示的な承認後のファイル変更

## 【違反した場合の対処】

操作を直ちに停止し、ユーザーに報告する：
「ルール違反ンの可能性があるため、操作を停止しました。続行するにはユーザーの明示的な許可が必要です。」
